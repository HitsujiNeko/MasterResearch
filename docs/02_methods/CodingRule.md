## 1. 命名規則
- **変数名・関数名**：英語で意味のある名前を付ける（例：`calculate_mean`、`data_frame`）。
- **クラス名**：単語の先頭を大文字にしたキャメルケース（例：`DataAnalyzer`）。
- **定数**：全て大文字、単語間はアンダースコア（例：`MAX_COUNT`）。

## 2. インデント・空白
- インデントは**半角スペース4つ**。
- タブは使用しない。
- 演算子の前後、カンマの後にはスペースを入れる。

## 3. コメントの書き方
- **日本語で記述**し、簡潔かつ分かりやすく。
- 初心者にも理解できるように丁寧に説明する。
- コードの意図や処理内容、注意点を明記する。
- 関数やクラスには**ドキュメンテーションストリング（docstring）**を必ず付与。

### コメント例
```python
# データフレームから欠損値を除外する
cleaned_df = df.dropna()
```

## 4. 関数・クラス設計
- 1関数1責務を意識し、処理を分割する。
- 関数には必ず**docstring**を付け、引数・戻り値・処理内容を日本語で記述。
- クラスも同様にdocstringを付与。

### docstring例
```python
def calculate_mean(values):
    """
    平均値を計算する関数

    Args:
        values (list of float): 計算対象の数値リスト
    Returns:
        float: 平均値
    """
    return sum(values) / len(values)
```

## 5. コーディングの基本方針
- 情報の**正確性**と**一貫性**を重視する。
- 再現性のあるコードを書く。
- 例外処理（try-except）を適切に行う。

## 6. エラー処理
- 予期しないエラーに備え、適切な例外処理を実装する。
- エラーメッセージは日本語で分かりやすく記述する。

## 8. その他
- コードは定期的に見直し、リファクタリングを行う。
- 不要なコードや使われていない変数は削除する。
- ファイルの先頭には、作成者・作成日・概要を日本語でコメントとして記載する。

## 9. 実装前後チェックリスト（必須）

以下は、AI実装・手動実装のどちらでも**コミット前に必ず確認**する。

### 9.1 実装前チェック
- [ ] 関連ドキュメント（少なくとも `docs/02_methods/CodingRule.md` と対象手法仕様）を再確認した
- [ ] 入出力パスは相対パスで設計し、`pathlib.Path` を使う方針にした
- [ ] 関数責務を分割し、1関数1責務の構成を決めた

### 9.2 実装後チェック
- [ ] インデントが半角スペース4つで、タブを使っていない
- [ ] 変数名・関数名・定数名が命名規則に従っている
- [ ] 追加・変更した関数に日本語docstringを付与した
- [ ] 複雑な処理に日本語コメントを付与した（過不足なく簡潔に）
- [ ] 例外処理を実装し、エラーメッセージが日本語で理解しやすい
- [ ] 再現性に影響する設定（パス、乱数、閾値）を明示した
- [ ] 不要なコード・未使用変数・デバッグ出力を残していない

### 9.3 出力前チェック
- [ ] 変更ファイルを自分で読み返し、規約違反がないことを確認した
- [ ] 主要ファイルの静的エラー（LSP/構文エラー）を確認した
- [ ] 変更内容を説明する際、根拠となるファイルを明示できる状態にした
