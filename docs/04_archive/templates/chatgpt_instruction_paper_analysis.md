# ChatGPT用指示書：論文分析・構造化要約生成

> **このファイルの役割**  
> ChatGPTに論文PDFや論文情報を渡し、GitHub Copilotが読み取り可能な構造化要約Markdownを生成させるための指示書です。

---

## 📋 ChatGPTへの指示（コピー&ペースト用）

```
あなたは研究支援AIとして、学術論文を分析し、構造化された要約を作成します。

## 背景情報

私は「ベトナム主要都市を対象とした地表面温度と都市構造の関係性評価」に関する修士研究を行っています。

### Research Questions（RQ）
- **RQ1**: Landsat 8（30m）LSTと都市構造データを組み合わせた場合、どの説明変数がLSTに対して支配的か？
- **RQ2**: 都市構造パラメータとLSTの関係性は、空間集計単位や解析スケールの違いによってどのように変化するか？
- **RQ3**: 測量データが限定的な条件下でも、衛星データおよび公開データによりLST分布の説明はどの程度可能か？

### 主要な関心キーワード
- Land Surface Temperature (LST)
- Urban Heat Island (UHI)
- 都市構造パラメータ（建物密度、道路密度、緑被率、NDVI、NDBI等）
- 機械学習（特にRandom Forest）
- 途上国都市
- データ制約

---

## 注意事項

1. **客観性**: 論文の内容を事実ベースで記述し、評価は「本研究との関連性」セクションでのみ行う
2. **RQ対応**: 必ずRQ1-3との関連性を評価する
3. **定量情報**: 数値データは正確に記載（p値、R²、RMSE等）
4. **用語統一**: 
   - Land Surface Temperature → LST
   - Urban Heat Island → UHI
   - 都市構造パラメータ → Urban Structure Parameters
5. **日本語**: 基本的に日本語で記述（論文タイトル・著者名は原語）

- [ ] Markdown形式が正しいか

- [ ] 都市構造パラメータが明確に定義されているか
- [ ] 重要な数値データが含まれているか
- [ ] 本研究への示唆が具体的か

## 🔄 使用方法

### ステップ1: ChatGPTにこの指示書をコピー
1. 上記「ChatGPTへの指示」セクション全体をコピー
2. ChatGPT（プロジェクト: 修士研究）を開く
3. ペースト

### ステップ2: 論文情報を追加
```
【論文情報】を埋める
- タイトル: [実際の論文タイトル]
- 著者: [著者名]
- 年: [出版年]
- DOI: [DOI]

またはPDFを添付
```

### ステップ3: ChatGPTの出力を保存
1. ChatGPTが生成したMarkdownをコピー
2. `docs/04_archive/02_structured_summaries/S[番号]_[著者]_[年].md` として保存
   - 例: `S9_Zhang_2023.md`

### ステップ4: GitHub Copilotに統合を依頼
```
GitHub Copilotに依頼：
「S9_Zhang_2023.md の内容を papers_database.csv に追加し、
 previous_studies_report.md を更新してください」
```
